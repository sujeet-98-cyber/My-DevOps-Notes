
# 📘 Ansible 

---

## 🧠 What is Ansible?
**Ansible** is an open-source **IT automation** and **configuration management** tool. It helps you:
- Automate app deployment 🚀
- Manage server configurations 🖥️
- Orchestrate complex workflows 🔁

✅ It is **agentless** (uses SSH)
✅ Uses **YAML** files called **Playbooks**

---

## 🧰 Core Concepts

### 📂 1. Inventory
Defines **which machines** to manage.

```ini
[web]
web1.example.com
web2.example.com

[db]
db1.example.com
```

### 🧾 2. Playbook
A **YAML file** that describes what actions to perform.

```yaml
- name: Install Apache
  hosts: web
  become: true
  tasks:
    - name: Install httpd
      yum:
        name: httpd
        state: present
```

### 🧩 3. Modules
Used to **perform actions** (e.g., install packages, copy files). Examples: `yum`, `apt`, `copy`, `service`, `file`, `user`.

### 🧠 4. Roles
Used to **organize playbooks** into reusable units (tasks, vars, handlers, etc.).

```bash
roles/
├── webserver/
│   ├── tasks/
│   └── handlers/
```

### 🧮 5. Variables
Store dynamic values used in playbooks.

```yaml
vars:
  package_name: httpd
```

### 🔁 6. Handlers
Tasks that only run when notified (e.g., restart a service after change).

```yaml
handlers:
  - name: restart apache
    service:
      name: httpd
      state: restarted
```

### 🔄 7. Idempotency
Running the same playbook **multiple times won't change** the system if nothing has changed.

### 🧾 Common Commands

| Command                             | Description                          |
|------------------------------------|--------------------------------------|
| `ansible all -m ping`              | Ping all hosts in inventory          |
| `ansible-playbook site.yml`        | Run a playbook                       |
| `ansible-vault encrypt file.yml`   | Encrypt a file                       |
| `ansible-doc -l`                   | List all available modules           |
| `ansible-inventory --list`         | Show inventory content               |

## 🧪 Top 10 Ansible Interview Questions with Answers

---

## ❓ 1. What is Ansible and why is it useful?

**💬 Answer:**  
Ansible is a **configuration management and automation** tool. It is **agentless**, simple to use, and helps manage infrastructure using **YAML-based playbooks**. It's great for automating repetitive tasks.



## ❓ 2. How does Ansible work?

**💬 Answer:**  
Ansible connects to hosts over **SSH**, pulls an inventory file, and runs **modules** defined in **YAML playbooks** to apply configurations.



## ❓ 3. What is the difference between ad-hoc commands and playbooks?

**💬 Answer:**  
- 📌 Ad-hoc: Quick one-time commands (e.g., install a package).  
- 📘 Playbook: Full configuration or deployment scripts in YAML.

```bash
ansible all -m ping       # Ad-hoc command
ansible-playbook site.yml # Playbook
```



## ❓ 4. What are Ansible modules?

**💬 Answer:**  
Modules are the **building blocks** of Ansible. They perform specific tasks like managing files, installing packages, or controlling services.

Examples: `copy`, `yum`, `apt`, `service`, `file`, `user`, `shell`.



## ❓ 5. What is an inventory file?

**💬 Answer:**  
The inventory file contains the **list of target machines** that Ansible will manage. It can be static or dynamic.

```ini
[web]
192.168.1.10
```



## ❓ 6. What are roles in Ansible?

**💬 Answer:**  
Roles are a way to **structure your playbooks** into reusable components. A role includes tasks, variables, handlers, files, and templates organized under a directory.



## ❓ 7. How is Ansible different from Puppet or Chef?

**💬 Answer:**  
- Ansible is **agentless** (uses SSH)
- Uses **YAML** instead of custom DSLs
- Easier to learn and use
- No need for a central server (can use Ansible Tower optionally)



## ❓ 8. What is a handler in Ansible?

**💬 Answer:**  
Handlers are tasks that run **only when notified** by another task (e.g., restart service if config file changes).



## ❓ 9. What is idempotency in Ansible?

**💬 Answer:**  
Ansible ensures that no matter how many times you run a playbook, the result is the **same**. No repeated or duplicate changes unless something changes.



## ❓ 10. How do you secure sensitive data in Ansible?

**💬 Answer:**  
Use **Ansible Vault** to encrypt variables or entire files.

```bash
ansible-vault encrypt secrets.yml
```
