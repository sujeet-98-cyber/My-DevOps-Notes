# 🐳 What is Docker?

Docker is an open-source platform designed to **create**, **deploy**, and **run applications** using **containers**.

---

## 🔧 How Docker Works

- Docker uses containers to run applications on the host operating system.
- It allows applications to share the same **Linux kernel** as the host system instead of creating a full virtual OS.
- ✅ This makes it **lightweight**, **fast**, and **efficient** compared to virtual machines.


## 💻 Compatibility

- Docker can be installed on **Linux**, **Windows**, and **macOS**.
- However, the **Docker Engine runs natively only on Linux** distributions.
- On Windows/macOS, Docker uses a lightweight VM (like **Hyper-V** or **WSL2**) to simulate a Linux environment.


## 📌 Key Facts about Docker

- 📝 **Written in**: Go (Golang)
- 🧩 **Virtualization Type**: OS-level (containerization)
- 💬 **Solves the issue**: "It works on my machine but not in production"



## 🗓️ History

- **First released**: March 2013  
- **Developed by**: Solomon Hykes and Sebastien Pahl

---

## 🐳 Docker vs Virtual Machine (VM) — Resource Usage 💡

### ⚙️ Virtual Machine (VM)

- A VM is a **full OS running on virtualized hardware**.
- Resources (CPU, RAM) are **reserved** at creation.
- ❌ Not flexible — cannot exceed its assigned resources.

### 🐳 Docker Container

- Docker containers **share the host OS kernel**.
- Containers **run as isolated processes**.
- ✅ Uses **only the required memory** dynamically.
- ✅ **Releases unused memory** after execution.
- ✅ Lightweight and efficient compared to VMs.

## 🐳 Docker Architecture – Explained

![Docker Architecture](https://github.com/akash08-ak/AWS-Project-Setup/blob/main/Interview-Preparation/Images/Docker%20-%20Architecture.png)

### 🔸 1. Docker Client

- Interface where commands are executed by the user  
  e.g., `docker build`, `docker run`, etc.
- Sends commands to Docker Daemon via **REST API**

### 🔸 2. Docker Host

- Runs the **Docker Daemon (dockerd)**
- Responsible for:
  - 🏗️ Building images
  - 📦 Creating containers
  - 📥 Pulling / 📤 Pushing images to/from registries
  - 📁 Managing volumes and 🔌 networks

### 🔸 3. Docker Registry (Repository)

- Stores Docker Images
- Can be:
  - 🌐 Public (e.g., Docker Hub)
  - 🔒 Private (e.g., AWS ECR, Harbor, GitHub Container Registry)


## 🔄 Docker Command Flow Examples

### ✅ 1. `docker build ubuntu`

- 📌 **Action**: Builds a Docker image  
- ⚙️ **Executed by**: Docker Daemon  
- 📦 **Result**: Image created and stored locally



### ✅ 2. `docker pull nginx`

- 📌 **Action**: Pulls an image from Docker Registry  
- ⚙️ **Executed by**: Docker Daemon  
- 📥 **Result**: Image downloaded to Docker Host



### ✅ 3. `docker run ubuntu`

- 📌 **Action**: Runs a container using Ubuntu image  
- ⚙️ **Executed by**: Docker Daemon  
- 🚀 **Result**: Container created and started



### ✅ 4. `docker push tomcat`

- 📌 **Action**: Pushes a local image to Docker Registry  
- ⚙️ **Executed by**: Docker Daemon  
- 📤 **Result**: Image uploaded to Registry

# 🐳 Docker Interview Questions & Answers 

---

## 1️⃣ What is Docker? 🐋

**Answer:**  
Docker is an open-source platform that automates the deployment, scaling, and management of applications using containerization. Containers package the application and its dependencies together, ensuring consistency across environments.



## 2️⃣ How is Docker different from a virtual machine (VM)? ⚙️

**Answer:**

| Feature         | Docker Container 🐳 | Virtual Machine 💻 |
|----------------|---------------------|--------------------|
| OS Overhead    | Shares host OS      | Full guest OS      |
| Startup Time   | Seconds             | Minutes            |
| Performance    | Lightweight & fast  | Heavy & slow       |
| Resource Usage | Dynamic             | Pre-allocated      |



## 3️⃣ What are Docker Images and Containers? 📦

**Answer:**  
- **Docker Image**: A read-only template used to create containers.  
- **Docker Container**: A running instance of a Docker image, isolated from the host system.



## 4️⃣ What is the difference between CMD and ENTRYPOINT in Dockerfile? 📝

**Answer:**  
- **CMD**: Sets default arguments that can be overridden at runtime.  
- **ENTRYPOINT**: Sets a fixed executable that always runs.

**Example:**  
If `ENTRYPOINT` is `["python3"]` and `CMD` is `["app.py"]`, then the final command executed is: `python3 app.py`



## 5️⃣ What is a Docker Volume? Why use it? 💾

**Answer:**  
A Docker Volume is a storage mechanism used to persist data generated by and used by Docker containers. Volumes help:

- Retain data even if the container is deleted  
- Share data between multiple containers  
- Improve I/O performance compared to bind mounts



## 6️⃣ What is Docker Compose? 🧩

**Answer:**  
Docker Compose is a tool used for defining and running multi-container Docker applications. You define the configuration in a `docker-compose.yml` file, making it easy to manage complex apps.

**Benefits:**  
- Easy setup of multiple containers  
- Shared networks and volumes  
- Simplified deployment for microservices



## 7️⃣ What is the difference between COPY and ADD in Dockerfile? 📁

**Answer:**

| Feature              | COPY | ADD |
|----------------------|------|-----|
| Local file copy      | ✅   | ✅  |
| Remote URL support   | ❌   | ✅  |
| Auto-extract TAR file| ❌   | ✅  |

📌 Use `COPY` for simple file copy tasks. Use `ADD` only when remote URLs or archive extraction is needed.



## 8️⃣ How do you optimize a Docker image? 🛠️

**Answer:**  
- Use minimal base images like `alpine`  
- Combine multiple `RUN` instructions with `&&`  
- Clean up unnecessary files and cache in the same layer  
- Use `.dockerignore` to avoid sending unused files  
- Apply multi-stage builds to separate build-time and runtime



## 9️⃣ How do you share data between Docker containers? 🔗

**Answer:**  
There are two common methods:

- **Volumes**: Shared persistent storage across containers  
- **Docker Networks**: Allows containers to discover and communicate via service names

**Example:**  
In Docker Compose, mount the same named volume in both containers to share data.



## 🔟 What is the role of Docker Registry? 🌐

**Answer:**  
A Docker Registry is a service for storing and distributing Docker images.

**Types:**  
- **Public**: Docker Hub  
- **Private**: AWS ECR, Harbor, GitHub Container Registry

**Commands:**  
- `docker push`: Uploads an image to a registry  
- `docker pull`: Downloads an image from a registry  
- `docker tag`: Tags an image for versioning
