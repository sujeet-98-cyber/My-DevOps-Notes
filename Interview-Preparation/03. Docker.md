# 🐳 What is Docker?

Docker is an open-source platform designed to **create**, **deploy**, and **run applications** using **containers**.

---

## 🔧 How Docker Works

- Docker uses containers to run applications on the host operating system.
- It allows applications to share the same **Linux kernel** as the host system instead of creating a full virtual OS.
- ✅ This makes it **lightweight**, **fast**, and **efficient** compared to virtual machines.


## 💻 Compatibility

- Docker can be installed on **Linux**, **Windows**, and **macOS**.
- However, the **Docker Engine runs natively only on Linux** distributions.
- On Windows/macOS, Docker uses a lightweight VM (like **Hyper-V** or **WSL2**) to simulate a Linux environment.


## 📌 Key Facts about Docker

- 📝 **Written in**: Go (Golang)
- 🧩 **Virtualization Type**: OS-level (containerization)
- 💬 **Solves the issue**: "It works on my machine but not in production"



## 🗓️ History

- **First released**: March 2013  
- **Developed by**: Solomon Hykes and Sebastien Pahl

---

## 🐳 Docker vs Virtual Machine (VM) — Resource Usage 💡

### ⚙️ Virtual Machine (VM)

- A VM is a **full OS running on virtualized hardware**.
- Resources (CPU, RAM) are **reserved** at creation.
- ❌ Not flexible — cannot exceed its assigned resources.

### 🐳 Docker Container

- Docker containers **share the host OS kernel**.
- Containers **run as isolated processes**.
- ✅ Uses **only the required memory** dynamically.
- ✅ **Releases unused memory** after execution.
- ✅ Lightweight and efficient compared to VMs.

## 🐳 Docker Architecture – Explained

![Docker Architecture](https://github.com/akash08-ak/AWS-Project-Setup/blob/main/Interview-Preparation/Images/Docker%20-%20Architecture.png)

### 🔸 1. Docker Client

- Interface where commands are executed by the user  
  e.g., `docker build`, `docker run`, etc.
- Sends commands to Docker Daemon via **REST API**

### 🔸 2. Docker Host

- Runs the **Docker Daemon (dockerd)**
- Responsible for:
  - 🏗️ Building images
  - 📦 Creating containers
  - 📥 Pulling / 📤 Pushing images to/from registries
  - 📁 Managing volumes and 🔌 networks

### 🔸 3. Docker Registry (Repository)

- Stores Docker Images
- Can be:
  - 🌐 Public (e.g., Docker Hub)
  - 🔒 Private (e.g., AWS ECR, Harbor, GitHub Container Registry)


## 🔄 Docker Command Flow Examples

### ✅ 1. `docker build ubuntu`

- 📌 **Action**: Builds a Docker image  
- ⚙️ **Executed by**: Docker Daemon  
- 📦 **Result**: Image created and stored locally



### ✅ 2. `docker pull nginx`

- 📌 **Action**: Pulls an image from Docker Registry  
- ⚙️ **Executed by**: Docker Daemon  
- 📥 **Result**: Image downloaded to Docker Host



### ✅ 3. `docker run ubuntu`

- 📌 **Action**: Runs a container using Ubuntu image  
- ⚙️ **Executed by**: Docker Daemon  
- 🚀 **Result**: Container created and started



### ✅ 4. `docker push tomcat`

- 📌 **Action**: Pushes a local image to Docker Registry  
- ⚙️ **Executed by**: Docker Daemon  
- 📤 **Result**: Image uploaded to Registry
