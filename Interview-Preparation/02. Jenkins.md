CI/CD is a method used in software development. It helps to build, test, and deliver code faster and more reliably.

CI (Continuous Integration):
Whenever a developer writes new code, it is automatically checked and added (integrated) with the existing code.
👉 This helps catch errors early.

CD (Continuous Delivery):
The code that passes all checks is automatically prepared for release.
👉 It’s ready to go live anytime with just one click.

CD (Continuous Deployment):
The code is not just prepared, but also automatically deployed to production (live) without manual steps.
👉 Changes reach users quickly and frequently.

🛠️ Jenkins and CI/CD – Simple Overview
Jenkins is a free and open-source tool written in Java. It runs on Windows, macOS, and Linux-based systems. It's widely used in software development for automating tasks like building, testing, and deploying code.

Originally called Hudson, Jenkins was created by Sun Microsystems in 2004. Later, when Oracle took over Sun, the project was renamed to Jenkins.

Jenkins supports Continuous Integration and Continuous Delivery (CI/CD), which means:

When developers write and push code, Jenkins automatically integrates everyone's work.

It then builds, tests, and deploys the software.

This makes the development process faster and helps catch bugs early, so they can be fixed quickly.

Jenkins is a powerful tool that helps teams deliver software quickly, reliably, and efficiently across almost any platform.



🔄 Jenkins Workflow (CI/CD Pipeline)
✅ Plugins Integration:
Jenkins supports various plugins like Git, Maven, Selenium, and Artifactory to extend its functionality.

📥 Code Checkout:
When a developer pushes code to GitHub, Jenkins automatically pulls the latest code from the repository.

🛠️ Build Stage:
Jenkins sends the pulled code to Maven to build the application.

🧪 Testing Stage:
After a successful build, Jenkins passes the code to Selenium for automated testing.

🗂️ Artifact Management:
Once testing is complete, Jenkins can send the final build to Artifactory or another artifact repository for storage or deployment.

🚀 Deployment (Optional):
Jenkins can also be used to deploy the application to a server or cloud environment as needed.
