# 🚀 CI/CD & Jenkins – Simple and Complete Overview

---

## 🔁 What is CI/CD?

![CI/CD](https://github.com/akash08-ak/AWS-Project-Setup/blob/main/Interview-Preparation/Images/CICD%20-%20Pipeline.png)

CI/CD is a method used in software development. It helps to **build, test, and deliver code faster and more reliably**.


### 🔧 CI – Continuous Integration
- Whenever a developer writes new code, it is **automatically checked and integrated** with existing code.
- 👉 Helps **catch errors early** in the development process.


### 📦 CD – Continuous Delivery
- Code that passes all checks is **automatically prepared for release**.
- 👉 Ready to go live **anytime with just one click**.


### 🚀 CD – Continuous Deployment
- Code is **automatically deployed to production** (live) without manual steps.
- 👉 Changes reach users **quickly and frequently**.

---
## 🛠️ Jenkins and CI/CD – Simple Overview

Jenkins is a **free and open-source automation tool**, written in Java.  
It runs on **Windows, macOS, and Linux** systems and automates tasks like:

- Building
- Testing
- Deploying code

### 📖 History
- Originally called **Hudson**, created by **Sun Microsystems** in 2004.
- Later renamed to **Jenkins** after Oracle acquired Sun.


### ⚙️ Jenkins Supports CI/CD
- When developers **push code**, Jenkins automatically:
  - Integrates everyone's work
  - Builds, tests, and deploys the application

✅ This makes development **faster**, more **reliable**, and **bug-free**.

---

## 🔄 Jenkins Workflow (CI/CD Pipeline)

1. ✅ **Plugins Integration**  
   Jenkins supports plugins like:
   - Git  
   - Maven  
   - Selenium  
   - Artifactory

2. 📥 **Code Checkout**  
   - Jenkins pulls the latest code from GitHub after a developer push

3. 🛠️ **Build Stage**  
   - Jenkins sends code to Maven to build the application

4. 🧪 **Testing Stage**  
   - Jenkins uses Selenium for **automated testing**

5. 🗂️ **Artifact Management**  
   - Final build is sent to Artifactory (or any repo)

6. 🚀 **Deployment (Optional)**  
   - Jenkins can deploy to server or cloud environments

## 🔹What is Single-node Jenkins?

![Single-node Jenkins](https://github.com/akash08-ak/AWS-Project-Setup/blob/main/Interview-Preparation/Images/Single-node%20Jenkins%20Architecture.png)

🧱 **One machine = Jenkins + Job Runner**  
Jenkins is installed on **one server or computer**.

That same machine runs:
- Jenkins Web UI  
- All jobs: Build, Test, Deploy

### ✅ Simple Setup
- Ideal for **small projects**, **testing**, or **learning**  
- Easy to install and manage

### ❌ Limitations
- Limited parallel job execution  
- **No scalability**  
- If server fails, Jenkins stops working

💡 **Example**:  
You install Jenkins on your **laptop** and run all pipelines from there — no extra machines.

## 🔹What is Master-Slave (Controller-Agent) Jenkins?

![Master-Agent Jenkins](https://github.com/akash08-ak/AWS-Project-Setup/blob/main/Interview-Preparation/Images/Master-Agent%20(Controller-Agent)%20Jenkins%20Architecture.png)

🧱 **One controller (master) + multiple agents (slaves)**



### 🧠 Controller (Master):
- Controls the pipeline  
- Assigns jobs to agents  
- Displays logs, results, and job status



### ⚙️ Agents (Slaves):
- Other machines connected to the controller  
- Perform actual work: build, test, deploy  
- Can run on **Windows**, **Linux**, or **cloud** machines



### ✅ Scalable Setup
- Run multiple jobs in **parallel**  
- Use **different environments** (Windows/Linux)  
- Perfect for **teams & large projects**



### ❌ Slightly Complex to Set Up
- Requires configuring agents (via SSH, JNLP, cloud plugins, etc.)

💡 **Example**:
- Jenkins controller runs on a **main server**  
- Agents:
  - 1 **Linux** server for **Java jobs**  
  - 1 **Windows** server for **.NET jobs**

👉 Controller **assigns jobs**, and **agents execute them**.
