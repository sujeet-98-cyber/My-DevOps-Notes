# 🚀 CI/CD & Jenkins – Simple and Complete Overview

---

## 🔁 What is CI/CD?

![CI/CD](https://github.com/akash08-ak/AWS-Project-Setup/blob/main/Interview-Preparation/Images/CICD%20-%20Pipeline.png)

CI/CD is a method used in software development. It helps to **build, test, and deliver code faster and more reliably**.


### 🔧 CI – Continuous Integration
- Whenever a developer writes new code, it is **automatically checked and integrated** with existing code.
- 👉 Helps **catch errors early** in the development process.


### 📦 CD – Continuous Delivery
- Code that passes all checks is **automatically prepared for release**.
- 👉 Ready to go live **anytime with just one click**.


### 🚀 CD – Continuous Deployment
- Code is **automatically deployed to production** (live) without manual steps.
- 👉 Changes reach users **quickly and frequently**.

---
## 🛠️ Jenkins and CI/CD – Simple Overview

Jenkins is a **free and open-source automation tool**, written in Java.  
It runs on **Windows, macOS, and Linux** systems and automates tasks like:

- Building
- Testing
- Deploying code

### 📖 History
- Originally called **Hudson**, created by **Sun Microsystems** in 2004.
- Later renamed to **Jenkins** after Oracle acquired Sun.


### ⚙️ Jenkins Supports CI/CD
- When developers **push code**, Jenkins automatically:
  - Integrates everyone's work
  - Builds, tests, and deploys the application

✅ This makes development **faster**, more **reliable**, and **bug-free**.

---

## 🔄 Jenkins Workflow (CI/CD Pipeline)

1. ✅ **Plugins Integration**  
   Jenkins supports plugins like:
   - Git  
   - Maven  
   - Selenium  
   - Artifactory

2. 📥 **Code Checkout**  
   - Jenkins pulls the latest code from GitHub after a developer push

3. 🛠️ **Build Stage**  
   - Jenkins sends code to Maven to build the application

4. 🧪 **Testing Stage**  
   - Jenkins uses Selenium for **automated testing**

5. 🗂️ **Artifact Management**  
   - Final build is sent to Artifactory (or any repo)

6. 🚀 **Deployment (Optional)**  
   - Jenkins can deploy to server or cloud environments

## 🔹What is Single-node Jenkins?

![Single-node Jenkins](https://github.com/akash08-ak/AWS-Project-Setup/blob/main/Interview-Preparation/Images/Single-node%20Jenkins%20Architecture.png)

🧱 **One machine = Jenkins + Job Runner**  
Jenkins is installed on **one server or computer**.

That same machine runs:
- Jenkins Web UI  
- All jobs: Build, Test, Deploy

### ✅ Simple Setup
- Ideal for **small projects**, **testing**, or **learning**  
- Easy to install and manage

### ❌ Limitations
- Limited parallel job execution  
- **No scalability**  
- If server fails, Jenkins stops working

💡 **Example**:  
You install Jenkins on your **laptop** and run all pipelines from there — no extra machines.

## 🔹What is Master-Slave (Controller-Agent) Jenkins?

![Master-Agent Jenkins](https://github.com/akash08-ak/AWS-Project-Setup/blob/main/Interview-Preparation/Images/Master-Agent%20(Controller-Agent)%20Jenkins%20Architecture.png)

🧱 **One controller (master) + multiple agents (slaves)**



### 🧠 Controller (Master):
- Controls the pipeline  
- Assigns jobs to agents  
- Displays logs, results, and job status



### ⚙️ Agents (Slaves):
- Other machines connected to the controller  
- Perform actual work: build, test, deploy  
- Can run on **Windows**, **Linux**, or **cloud** machines



### ✅ Scalable Setup
- Run multiple jobs in **parallel**  
- Use **different environments** (Windows/Linux)  
- Perfect for **teams & large projects**



### ❌ Slightly Complex to Set Up
- Requires configuring agents (via SSH, JNLP, cloud plugins, etc.)

💡 **Example**:
- Jenkins controller runs on a **main server**  
- Agents:
  - 1 **Linux** server for **Java jobs**  
  - 1 **Windows** server for **.NET jobs**

👉 Controller **assigns jobs**, and **agents execute them**.

# 💼 Jenkins Top 10 Interview Questions (3 Years Experience)

---

## 1️⃣ What is Jenkins?

🤖 **Jenkins** is an open-source CI/CD automation server.  
It supports building, testing, and deploying code using plugins.



## 2️⃣ What are Jenkins Pipelines?

📦 **Jenkins Pipeline** is a scripted workflow that automates CI/CD.

**Types:**
- Declarative (simple, recommended)
- Scripted (Groovy-based, flexible)

💡 **Example:**
```groovy
pipeline {
  agent any
  stages {
    stage('Build') { steps { echo 'Building...' } }
    stage('Test')  { steps { echo 'Testing...' } }
    stage('Deploy'){ steps { echo 'Deploying...' } }
  }
}
```
## 3️⃣ Difference Between Freestyle and Pipeline Projects?

| Feature        | Freestyle Job     | Pipeline Job           |
|----------------|-------------------|--------------------------|
| GUI vs Code    | GUI-based         | Code in Jenkinsfile      |
| Flexibility    | Limited           | High                     |
| SCM            | Manual            | Version-controlled       |



## 4️⃣ How to Add an Agent Node?

🖥️ **Steps to add agent node in Jenkins**:

1. Go to **Manage Jenkins → Manage Nodes**
2. Click **New Node**
3. Enter node **name**, set **usage** and **labels**
4. Choose **launch method** (SSH or JNLP)
5. Download and run `agent.jar` on the node machine



## 5️⃣ What is a Jenkinsfile?

📄 **Jenkinsfile** is a text file stored in source control that defines your CI/CD pipeline as code.

✅ **Benefits**:
- Version-controlled
- Reusable across environments
- Easy team collaboration



## 6️⃣ How to Trigger Jenkins Jobs?

🚀 **Ways to trigger jobs in Jenkins**:
- Manually using **Build Now**
- **SCM webhook** (e.g., GitHub push)
- **CRON syntax** for scheduled jobs
- **API trigger** using REST endpoint
- Triggered by another Jenkins job



## 7️⃣ Jenkins Git Integration?

🔗 **Steps to integrate Git with Jenkins**:

1. Install **Git plugin**
2. In job config, add **Git repository URL**
3. Set **branch** to build (e.g., `main`)
4. Add **credentials** if needed
5. Configure **GitHub webhook** for auto-triggering on push



## 8️⃣ What is Jenkins Blue Ocean?

🌊 **Blue Ocean** is a modern Jenkins UI plugin designed for Pipelines:

- Visual editor for pipelines
- Graphical view of stages and logs
- Better debugging and UX



## 9️⃣ How Do You Manage Credentials?

🔐 **Using Credentials Plugin**:

**Steps**:
1. Go to **Manage Jenkins → Credentials**
2. Add credentials (e.g., secret text, username/password)

**Use in pipeline**:
```groovy
withCredentials([usernamePassword(credentialsId: 'my-creds', usernameVariable: 'USER', passwordVariable: 'PASS')]) {
    sh 'echo $USER && echo $PASS'
}
```
## 🔟 How to Handle Failures in Jenkins Pipelines?

💥 **Best Practices to Handle Failures:**

- ✅ Use `try-catch` blocks to catch and handle errors gracefully
- 🔁 Use `retry` block for steps that may fail intermittently
- 📦 Use `post` block for cleanup or notifications (e.g., always, success, failure)
- ❗ Mark builds as **unstable** or **failed** using `currentBuild.result`
- 📧 Send notifications (Slack, Email, etc.) when a stage fails

💡 **Example Pipeline Snippet**:
```groovy
pipeline {
  agent any

  stages {
    stage('Build') {
      steps {
        echo 'Building...'
      }
    }

    stage('Deploy') {
      steps {
        retry(2) {
          sh './deploy.sh'
        }
      }
    }
  }

  post {
    failure {
      echo 'Build failed. Sending notification...'
      // You can add email or Slack notification steps here
    }
    always {
      echo 'Cleaning up...'
    }
  }
}
```
